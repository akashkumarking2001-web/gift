## 1. PRODUCT OVERVIEW & VISION

### 1.1 What We're Building

A marketplace where emotional moments become interactive digital experiences:
- **For Gifters:** Easy-to-use editor to customize professional templates
- **For Recipients:** Immersive, mobile-optimized gift experiences
- **For Business:** Scalable SaaS model with minimal operational overhead

### 1.2 Target Market

**Primary Audience:**
- Age: 18-35 years
- Relationship status: In relationships, close friendships
- Tech-savvy, comfortable with digital gifting
- Geography: Initially India, expandable globally

**Use Cases:**
- Valentine's Day expressions
- Birthday surprises
- Anniversary celebrations
- Apologies and reconciliations
- Friendship appreciation
- Special announcements

### 1.3 Business Model

**Revenue Streams:**
1. Individual template sales (â‚¹50-â‚¹299)
2. Bundle packages (3 templates for â‚¹99 - Valentine's Special)
3. Category bundles (All Birthday templates, etc.)
4. Future: Subscription model (â‚¹299/month - unlimited access)

**Payment Method:** UPI (Manual Verification)
- User makes payment via UPI app
- Submits transaction ID
- Admin manually verifies and approves
- Account activated within 2 hours

### 1.4 Success Metrics

**MVP Success:**
- 100 paying users in first month
- 500+ gifts created
- 70%+ purchase completion rate
- < 5% refund rate
- 4.5+ star rating (user feedback)

```typescript
// app/dashboard/editor/[id]/page.tsx

export default function TemplateEditor({ params }: { params: { id: string } }) {
  const [gift, setGift] = useState<Gift | null>(null);
  const [currentPage, setCurrentPage] = useState(0);
  const [saving, setSaving] = useState(false);

  // Auto-save effect
  useEffect(() => {
    const timer = setInterval(() => {
      if (gift?.hasChanges) {
        saveGift();
      }
    }, 30000); // 30 seconds

    return () => clearInterval(timer);
  }, [gift]);

  // Calculate completion status
  const getPageStatus = (page: PageData) => {
    const requiredFields = page.requiredFields || [];
    const filledFields = requiredFields.filter(field => 
      page.content[field] && page.content[field].length > 0
    );
    
    if (filledFields.length === 0) return 'not-started';
    if (filledFields.length === requiredFields.length) return 'complete';
    return 'in-progress';
  };

  const updatePage = (pageIndex: number, updates: Partial<PageData>) => {
    setGift(prev => ({
      ...prev!,
      pages: prev!.pages.map((p, i) => 
        i === pageIndex ? { ...p, ...updates } : p
      ),
      hasChanges: true
    }));
  };

  const generateLink = async () => {
    const link = await generateGiftLink(gift!);
    // Show modal with link
    setGeneratedLink(link);
  };

  return (
    <EditorLayout>
      <PageList
        pages={gift?.pages || []}
        currentPage={currentPage}
        onPageSelect={setCurrentPage}
        getStatus={getPageStatus}
      />
      
      <PreviewPane
        gift={gift}
        currentPage={currentPage}
      />
      
      <EditingPanel
        page={gift?.pages[currentPage]}
        onUpdate={(updates) => updatePage(currentPage, updates)}
      />
      
      <EditorToolbar
        onSave={saveGift}
        onGenerateLink={generateLink}
        saving={saving}
        canGenerate={isAllPagesComplete()}
      />
    </EditorLayout>
  );
}
```

**Frontend:**

- **Styling:** Tailwind CSS + Custom CSS
- **Animations:** Framer Motion (primary), CSS animations
- **State Management:** Zustand (lightweight)
- **Forms:** React Hook Form + Zod validation
- **Images:** Next/Image + Sharp
- **Icons:** Lucide React
- **Drag & Drop:** @dnd-kit or react-beautiful-dnd
- **Confetti:** canvas-confetti
- **Characters:** Lottie React (for animated characters)
- **Music:** Howler.js
- **QR Code:** qrcode.react
- **Rich Text:** Tiptap or Lexical
- **Date:** date-fns

**Backend:**
- **Platform:** Supabase
  - Authentication (email/password)
  - PostgreSQL database
  - Storage (user photos, template assets)
  - Real-time subscriptions (admin panel)
  - Row Level Security (RLS)

**Deployment:**
- **Hosting:** Vercel (optimized for Next.js)
- **Domain:** Custom domain with HTTPS
- **CDN:** Vercel Edge Network (automatic)
- **Analytics:** Vercel Analytics + Google Analytics

### 6.2 Database Schema

```sql
-- Enable UUID extension
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- Users table
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  email VARCHAR(255) UNIQUE NOT NULL,
  full_name VARCHAR(255) NOT NULL,
  mobile VARCHAR(20),
  password_hash VARCHAR(255) NOT NULL,
  status VARCHAR(20) DEFAULT 'pending', -- pending, approved, rejected
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Templates table (master list)
CREATE TABLE templates (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  name VARCHAR(255) NOT NULL,
  slug VARCHAR(255) UNIQUE NOT NULL,
  category VARCHAR(100) NOT NULL, -- valentine, birthday, anniversary, apology, fun
  description TEXT,
  preview_video_url TEXT,
  thumbnail_url TEXT,
  page_count INTEGER NOT NULL,
  complexity VARCHAR(20) DEFAULT 'medium', -- low, medium, high
  price_inr INTEGER NOT NULL,
  sale_price_inr INTEGER,
  is_active BOOLEAN DEFAULT true,
  features JSONB, -- Array of feature strings
  page_config JSONB NOT NULL, -- Full page configuration
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Purchases table
CREATE TABLE purchases (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  transaction_id VARCHAR(255) UNIQUE NOT NULL, -- UTR from UPI
  amount_inr INTEGER NOT NULL,
  payment_status VARCHAR(20) DEFAULT 'pending', -- pending, verified, rejected
  templates_included JSONB NOT NULL, -- Array of template IDs
  purchase_type VARCHAR(50) DEFAULT 'individual', -- individual, bundle
  created_at TIMESTAMP DEFAULT NOW(),
  verified_at TIMESTAMP,
  verified_by UUID REFERENCES users(id), -- Admin who verified
  rejection_reason TEXT
);

-- User templates (junction table)
CREATE TABLE user_templates (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  template_id UUID REFERENCES templates(id) ON DELETE CASCADE,
  purchase_id UUID REFERENCES purchases(id),
  created_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(user_id, template_id)
);

-- Gifts table (created instances)
CREATE TABLE gifts (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  template_id UUID REFERENCES templates(id),
  gift_uuid VARCHAR(36) UNIQUE NOT NULL, -- Public shareable UUID
  gift_name VARCHAR(255) DEFAULT 'My Gift',
  gift_data JSONB NOT NULL, -- All customized content
  recipient_name VARCHAR(255),
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW(),
  view_count INTEGER DEFAULT 0,
  last_viewed_at TIMESTAMP
);

-- Gift photos table
CREATE TABLE gift_photos (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  gift_id UUID REFERENCES gifts(id) ON DELETE CASCADE,
  page_number INTEGER NOT NULL,
  photo_url TEXT NOT NULL,
  caption TEXT,
  display_order INTEGER DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Admin users table (separate from regular users)
CREATE TABLE admin_users (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  full_name VARCHAR(255) NOT NULL,
  role VARCHAR(50) DEFAULT 'admin', -- admin, super_admin
  created_at TIMESTAMP DEFAULT NOW(),
  last_login_at TIMESTAMP
);

-- Analytics events (optional)
CREATE TABLE analytics_events (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  event_type VARCHAR(100) NOT NULL, -- page_view, purchase, gift_created, gift_viewed
  user_id UUID REFERENCES users(id),
  gift_id UUID REFERENCES gifts(id),
  metadata JSONB,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Create indexes for performance
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_status ON users(status);
```

### 6.3 Supabase Storage Buckets

```
Buckets Configuration:

1. user-photos (Public read, authenticated write)
   - Stores user-uploaded photos for gifts
   - Path structure: /{gift_id}/photo_{timestamp}.jpg
   - Max file size: 5MB
   - Allowed types: image/jpeg, image/png, image/webp

2. template-previews (Public read, admin write only)
   - Stores template preview videos and thumbnails
   - Path structure: /{template_id}/preview.mp4, thumbnail.jpg
   - No size limit (admin controlled)

3. template-assets (Public read, admin write only)
   - Stores template-specific assets
   - Path structure: /{template_id}/character.svg, background.jpg, audio.mp3
```

**Storage Policies:**

```sql
-- User photos: Users can only upload to their own gifts
CREATE POLICY "Users upload own gifts"
ON storage.objects FOR INSERT
TO authenticated
WITH CHECK (
  bucket_id = 'user-photos' AND
  (storage.foldername(name))[1] IN (
    SELECT id::text FROM gifts WHERE user_id = auth.uid()
  )
);

-- User photos: Public read
CREATE POLICY "Public read user photos"
ON storage.objects FOR SELECT
TO public
USING (bucket_id = 'user-photos');

-- User photos: Users can delete own
CREATE POLICY "Users delete own photos"
ON storage.objects FOR DELETE
TO authenticated
USING (
  bucket_id = 'user-photos' AND
  (storage.foldername(name))[1] IN (
    SELECT id::text FROM gifts WHERE user_id = auth.uid()
  )
);

-- Template previews/assets: Public read
CREATE POLICY "Public read templates"
ON storage.objects FOR SELECT
TO public
USING (bucket_id IN ('template-previews', 'template-assets'));

-- Template previews/assets: Admin write only
CREATE POLICY "Admin write templates"
ON storage.objects FOR INSERT
TO authenticated
WITH CHECK (
  bucket_id IN ('template-previews', 'template-assets') AND
  EXISTS (SELECT 1 FROM admin_users WHERE id = auth.uid())
);
```

### 6.4 Row Level Security (RLS) Policies

```sql
-- Users table: Users can read own data
CREATE POLICY "Users read own"
ON users FOR SELECT
TO authenticated
USING (id = auth.uid());

-- Users table: Users can update own data
CREATE POLICY "Users update own"
ON users FOR UPDATE
TO authenticated
USING (id = auth.uid());

-- Templates table: Everyone can read active templates
CREATE POLICY "Public read active templates"
ON templates FOR SELECT
TO public
USING (is_active = true);

-- Purchases table: Users can read own purchases
CREATE POLICY "Users read own purchases"
ON purchases FOR SELECT
TO authenticated
USING (user_id = auth.uid());

-- User templates: Users can read own
CREATE POLICY "Users read own templates"
ON user_templates FOR SELECT
TO authenticated
USING (user_id = auth.uid());

-- Gifts table: Users can CRUD own gifts
CREATE POLICY "Users read own gifts"
ON gifts FOR SELECT
TO authenticated
USING (user_id = auth.uid());

CREATE POLICY "Users create gifts"
ON gifts FOR INSERT
TO authenticated
WITH CHECK (user_id = auth.uid());

CREATE POLICY "Users update own gifts"
ON gifts FOR UPDATE
TO authenticated
USING (user_id = auth.uid());

CREATE POLICY "Users delete own gifts"
ON gifts FOR DELETE
TO authenticated
USING (user_id = auth.uid());

-- Gifts table: Public can read gift by UUID (for viewing)
CREATE POLICY "Public read gift by UUID"
ON gifts FOR SELECT
TO public
USING (gift_uuid IS NOT NULL);

-- Gift photos: Users can read photos of own gifts
CREATE POLICY "Users read own gift photos"
ON gift_photos FOR SELECT
TO authenticated
USING (
  gift_id IN (SELECT id FROM gifts WHERE user_id = auth.uid())
);

-- Gift photos: Users can insert photos to own gifts
CREATE POLICY "Users insert own gift photos"
ON gift_photos FOR INSERT
TO authenticated
WITH CHECK (
  gift_id IN (SELECT id FROM gifts WHERE user_id = auth.uid())
);

-- Gift photos: Users can delete photos from own gifts
CREATE POLICY "Users delete own gift photos"
ON gift_photos FOR DELETE
TO authenticated
USING (
  gift_id IN (SELECT id FROM gifts WHERE user_id = auth.uid())
);

-- Gift photos: Public can read photos (for public gift viewing)
CREATE POLICY "Public read gift photos"
ON gift_photos FOR SELECT
TO public
USING (
  gift_id IN (SELECT id FROM gifts WHERE gift_uuid IS NOT NULL)
);
```
